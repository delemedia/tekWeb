Base 2 - Det binære - "bits" forklaring ( 2-tal system )



Base 10 tal system – Er ikke det eneste tal system der eksistere.


Når man anvender en hver form for enhed - Som har en digitale format.


Digitale systemer – Bruger 2 tal systemer. Som hedder Base 2.



Ved Base 2 – Kaldes det Binære tal system


Ved det binære betyder - At man har 2 dele. Som kan være 0 eller 1.


Det kendes også som ( Slukket eller Tændt ) = Stik kontakt.


I en computer - Der kan 1 bit = Være slukket eller tændt.


Har betydning for at computerens interne sammensætning og IP netværk opsætninger.

Udregning af - ( Binær --> Decimal ):





BI = 2 - ( Er et 2 tal-system. )


Man regner i vægtens potens - Såsom: 2 ^ 4 = 16   Svarer til 2 * 2 * 2 * 2 = 16


Man tager fra det MINDST betydende cifre ( LSB )    Fra Højre side ad.

Til det MEST betydende cifre ( MSB )    På langs venstre side ad.


Man kan selv beslutte rækkefølgen. Men udregningen er det samme.



Det gør det lidt nemmere, at udregne / overblik i binære tal værdier.

Ved at indsætte "bits'ene" ind i en positions tabel.



Tabel eksempler:






Der bliver omregnet fra det her binære tal:  10110


Som eksempel - Fra Binære --> Decimal tal.





Det lille tal indikere ( index ).

At det er en Base 2 tal værdi - Der er angivet.



Visuel model - Udregning af værdien for:  11010





Resultatet er:  22    ( Base 10 forstand )


Fra Base 2:  ( 1 0 1 1 0 ) --> til Base 10 ( 22 )   De er det samme tal.


Det lille 10 tal ved 22 - Er for at indikere at det Base 10 format.


Det er 1 måde at regne fra:

Binære ( Base 2 ) til Decimal ( Base 10 ).




En anden måde / udregning…


Er at 'udpege' - Hvert bit værdi - Udfra positions tabellen struktur

Lidt mere direkte...




Det vil sige...


Udfra 10110 - Er der:


  • 0 – 1’ere.     = 0


  • 1 – 2’ere.     = 2


  • 1 – 4’ere.     = 4


  • 0 – 8’ere.     = 0


  • 1 – 16’ere.   = 16


Udregning = 2 + 4 + 16 = 22



Som eksempel - På alternativ udregning:





Det var den anden måde - Man regnet ( Fra Binære til Decimal. )




Udregning af - ( Decimal --> Binær ):


Hvis man skal regne omvendt?   Fra Decimal til Binære?


Når man mener Rest = 27 / 2 = 16,5


De 0,5 er rest = Det binære tal: 1 ( Ergo ulige )



Men f.eks 6 / 2 = 3


Der er det et ”hel tal” og det giver os = Det binære tal: 0. ( Ergo lige )






Der er taget et vilkårlig tal såsom: 27.


Hvor man dividere det omvendte resultat.


Toppen er det MEST betydende cifre.

Bunden er det MINDST betydende cifre.


Som man derefter lægger ned - I en lang 'horizontal' linje.








En anden måde / udregning…


Er at udregne det mere direkte...








Det er samme berening - Med at dividere hvert tal med 2.


27 / 2 = 13,5 = Rest på 0,5 = 1


Derefter...   6 / 2 = 3 Intet rest = 0


Det bliver derefter...   ulige = 1  og  lige = 0   Fra 27 --> 1 1 0 1 1



De næste eksempler - Er "Ekstra indhold."


Varieret Regne-arter af det Binære talsystem






Man regner med digits.   Bit  =  Digits fra Digits  ( 0 eller 1 )


Der er visse Algebra regler - I den her sammenhæng.


Såsom at:  ( +  og  +  giver  - )


1 + 1   Det er 0.   Men man skal huske - At 1 over i 'mente.'



Det her er addition reglerne:





1 + 1 = 10   1 i mente



Det her er en binære udregning af:   1011  +  101


Resultat bliver: 10000 bits --> ( 16 tal )



Gennemgang af udregningen:


  |-----------------------------|


Øverste bit-række:


8 4 2 1 - ( Positions tabel )

1 0 1 0 - ( Binære tallet )

8 0 2 1 = ( 8 + 2 + 1 ) = 11 - ( Decimal tallet )


  |-----------------------------|



  |-----------------------------|


Midterste bit-række:


4 2 1 - ( Positions tabel )

1 0 1 - ( Binære tallet )

4 0 1 = ( 4 + 1 ) = 5 - ( Decimal tallet )


  |-----------------------------|



  |-----------------------------|


Det samlet binære tal:


16 8 4 2 1 - ( Positions tabel )

 1 0 0 0 0 - ( Binære tallet )

1 6 0 0 0 0 = ( 16 + 0 + 0 + 0 + 0 ) = 16 - ( Decimal tallet )


  |-----------------------------|



Ekstra eksempel:





8 4 2 1 - ( Positions tabel )

1 1 1 1 - ( Binære tallet )

8 4 2 1 = ( 8 + 4 + 2 + 1 ) = 15 - ( Decimal tallet )








Det kan give mindre problemer - Når man regner med:  0 – 1 = 1


Men det gør det også...  Hvis man kun kan låne - '1 mente' fra det næste større bit.



Hvis man låner 1 bit. Bliver det fordoblet med 2 bit = 1 + 1


De tal man låner - Går med hinanden - Hvis der er flere end 1 bits.



Det her er subtraktion reglerne:










Når man ganger med decimal tal.


Der tager man det enkelte tal - og ganger igennem alle cifre.


Såsom:  121 * 35  =  Sidste 1 tal  * 5 = 5   og sidste 1 tal * 3 = 3


Derefter går man til næste tal position - I placering 2 - Og starter forfra.



Når 1’eren er klaret.  Skal man rykke op til 10’erene.


Derefter afkryser man med x - På den sidste placering.

Man flytter en plads frem til venstre.


Det samme med at gange med 100’erene.

Bare med 2 x x til sidst.





Det er sådan - Man ganger med almen 'decimal' store tal.


Men det bliver lidt på samme måde - Når man ganger med udreninger til bits.


Man ender med at:  1 * 1 = 1


Når det er 0 - Bliver alt ganget til = 0



Når man har gennemgået tallene.


Ligesom det ovenstående multiplikation - Tal eksempel.


Der lægger man - Derefter bits’ene sammen - Som i addition eksemplet.






Det bliver overlagt 5 * 11 = 55



 32  16   8   4   2   1 - ( Positions tabel )

    1   1   0   1   1   1 - ( Binære tallet )

 32  16   0   4   2   1 = ( 32 + 16 + 0 + 4 + 2 + 1 ) = 55 - ( Decimal tallet )








Siden 35 er et 2 cifre tal.


Skal man fokusere på de - 2 første største tal.


Går 35 op i 42?   -->   Ja, det gør det 1 gang.


35 / 42  =  42 – 35 = 7


( Vigtigst, at det skal gå op i 35 tallet )   Ikke omvendt.


Indikere i toppen med et 1 tal.

Der efterlader resultatet 7 i bunden.



Derefter hiver man de næste cifre ned...

Som er 3.   Der udregner - Om 35 går op i 73.


Det går 2 gange op i 73. = 35 + 35 = 70


Derefter hiver man det sidste 5 tal ned - Og udregner igen.






Det giver i alt = 121

Ved at dividere med et almen decimal tal.



Men det er samme udregnng struktur - Når man dividere med udreninger til bits.





8 4 2 1 - ( Positions tabel )

1 0 1 1 - ( Binære tallet )

8 0 2 1 = ( 8 + 0 + 2 + 1 ) = 11 - ( Decimal tallet )



Samme logik - Bliver overført til Bits / Binære udregninger.


At 101 skal dividere - Med de 3 første bits i udregningen.


Hvad gør man?   Når man ikke kan trække 1 0 1  /  0 0 1 1 ??   Siden 1 0 1 er større?


Siden man ikke kan trække 101 fra 0011 – Automatisk - Bliver det så 0 oppe i toppen.


Derefter afkrydser man 101 væk.

Derefter...  Er det de 2 øverste 11 - Som man så trækker ned i stedet for.


Ergo...  Bliver det 111 – 101   Det kan man godt trække fra.

Som der, giver et 1 i toppen.

  • A  =  10

  • B  =  11

  • C  =  12

  • D  =  13

  • E  =  14

  • F  =  15



Man regner med vægtens potens 16 ^ 2  =  16 * 16 = 256

Angående hvilken position tallet er placeret i.



A   3   8   =   8  har position 0.


Derfor bliver 16 opløftet i 0.


Eksempel:   16^0 = 1   16^0   = 1


Når et tal går op i potens med 0  Bliver det altid 1.


|--------------------------------------------------|


A   3   8   =   3  har position 1.


Derfor bliver 16 opløftet i 1.


Eksempel:   16^1 = 16  


A = 10.   Derefter er 16^2 = 256.

Abstrakt måde at forklare det på:


Procedural ( Imperative ) - Er på 'Hvordan' man struktur kode med.

Fremfor....

Declarative ( non-procedural ) - Der er på 'Hvad' man struktur kode efter.

    2.

Hvordan man skaber en Azure Database - MySQL



Klassens 'Korte' gennemgang:


Opret et MySQL DBMS - Azure:

  1.    Fuldfør Login proceduren:

  2.    I søgefeltet øverst på websitet - Indtastet: "Azure Database for MySQL flexible servers"

  3.    Klik på knappen - Create

  4.    Indtast "Ressource group" - Med et (Password) - Noget der kan huskes.

  5.    Alt. justere på "Compute + Storage" - Til behov og billigst beløb.

  6.    Indtast "Server name" med et unikt navn - Andre servere må ikke hedde det samme.

  7.    Vælg en region tæt på Dannmark - Såsom: "North Europe"

  8.    Indtast "Work Load" - Sæt den til "For development or hobby projects"

  9.    Indtast værdier for "Admin username" - "Password" - "Confirm password"

  10.    Placer checkmark på "Allow public access from any Azure service within Azure to this server."

  11.    Klik på "+ Add 0.0.0.0 - 255.255.255.255"    Så alle kan tilgå ens MySQL DBMS.

  12.    Klik på enten knappen:  "Create" eller "Review+Create


Derefter går Azure platformen - Igang med at deploy og oprette en MySQL DBMS


Det tag et par min - Derefter har man en MySQL til rådighed.




Alternativ noter:

Vælg i venstre panel:  Databases - Derefter:  SQL Database.


Vælg RDP ( Remote Desktop Protocol )

Vælges for få adgang til enheden - Igennem ens internet.

Visuel guide gennemgang:



Hvordan man skaber en MySQL DBMS - Azure:



Gå til websitet - Login med ens profil






Søg efter mysql --> Og Vælg - Azure database for MYSQL servers.






Derefter kan man vælge imellem 2 create buttons - For at starte processen.






Man kan vælge i ressource - Imellem en:  Flexible eller Single server.

Vælg en single server.

Klik på den blå knap – Create.






Udvælg den rette Ressource Group format.






Navngiv sin server.


Vælg om man vil - Har en backup løsning.


Vælg ens lokation – Hvor ens virtuelle server bliver oprettet henne.

Såsom: "North Europe"

Vælg hvad for en - Version man forholder sig til.


Vælg imellem forskellige Storage / Ydeevne ( performance ) muligheder.


Klik på Configure server.






I forhold til hvilken udgave - Af performance man vælger.  Desto bedre – Desto dyrere.


Det billigste format er:  Basic – 1 VCore


Ens storage er sat til 5GB – Den angivet plads man har.


Klik på Ok knappen









Ved Additional settings tab'en – Er det muligt at anvende Data encryption.






Det samme med at tildele Tags.






Hvis man prøver at lave maskinen for hurtigt - Validation warning.


Alting skal lige valideres først.






Under Basic tab'en:

Username + Passwords - Skal lige tilføjes.






Nu er det muligt - At vælge Review + Create.


Når vilkårene er blevet gennemtjekket. Kan man klikke på Create knappen igen.





Derefter bliver maskinen ’deployed’ + Created.






Man kan se at ens Deployment er færdig


Man kan få et overblik af detaljerne på ens enhed.


Klik på:  Go to Ressource.






Inde i Ressource page: Kan man se et overview - Af detajler på ens nyskabte virtuelle maskine.


I venstre panel – Har man en del yderligere valgmuligheder.


Der er det vigtigt - At man notere / kopiere ens server name + server admin login name.


Vigtigt - At kunne finde ens servernavn ID info.  Når man skal forbinde ens DBMS til en MySQL Database Workbench.






Hvis man går tilbage til Home page.  Kan man også vælge ens enhed herfra.






Hvordan man forbinder ens Azure DBMS --> MySQL Workbench:


Man åbner ens MySQL Workbench.


Hvis man ønsker et ny forbindelse – Klik på (+) tegnet.






Navngiv - Hvad ens MySQL forbindelse skal hedde.


  1. Tilfører ( Copy n Paste ) i Hostname. --> Servername.

  2. Tilfører ( Copy n Paste ) i Username --> Server admin login name


Klik på - Test Connection.




                    





Hvis den her fejl besked opstår?


Skal der justeres en Firewall indstilling inde i Azure platformen






Der skal justeres - Tilføjes / Bypasses en IP adresse.


Vælg i venstre panel - Connection Secuirty. -


Klik på:   + Add current IP address.


Er derefter tilføjet til en Firewall protokol i Azure.

Ved at tilpasse / overfører IP Adresser.

Klik på Save til sidst.






Hvis man får succesful update forbindelse.






Kan man klikke på - Test Connection igen.


Hvis det er muligt - Kan man indtaste et password - Er man godt kørende.

Vælg - Save password in vault. Hvis det er præferencen/sikkerhed.





Forbindelsen fungere.






Nu er der oprettet en database forbindelse.

Til ens Azure DBMS platform






For at arbejde i databasen - Klikker man på ens oprettet database forbindelse.






Nu kan vi:


Create Database - Skab en database


Use database - Brug en database


Create a table - Skab en tabel som indeholder ( Int = Tal / VARCHAR = Strings-Tekst )


Show tables - Vis tabellen.






INSERT INTO ( id, name, email, ) - Sætte variabler ind i "Tabellen"


VALUES ( id-værdi, name-værdi, email-værdi ) - Sætte værdier ind i "Tabellen"


SELECT * FROM - Man vælger at se ( Alt = * ) indholdet fra "Tabellen"
