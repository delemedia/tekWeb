What Is DNS?



DNS Servers og IP Adresser



DNS Definition:

Domain Name System (DNS) gør domæne-navne til IP-adresser.

Som browsere anvender til at hente internet-sider.

'DNS henter IP adressen til brugeren.'

Hver enhed der er tilsluttet internettet, har sin egen IP-adresse.

Som bruges af andre enheder til at lokalisere enheden.


DNS-servere gør det muligt for folk, at let indtaste:

"Normale ord / navne" i deres browsere.

Såsom Google.com = ( 8.8.8.8 )

Uden DNS ville man holde styr på en lang liste af IP-adresser.

For hver website man besøger.


Computere og forskellige enheder, der bruger internettet.

Er afhængige af IP-adresser.

Hvis man f.eks. indtaster et website i ens webbrowser,

kan den her URL, ”ikke i sig selv” bringe dig til websitet.

De her bogstaver kan ikke "læses" af de servere.


Serverne er dog i stand til at læse IP-adresser.




Hvad er en DNS Server?

En DNS-server er en computer med en database,

der indeholder de offentlige IP-adresser,

der er forbundet med navnene på de websteder,

en IP-adresse bringer en bruger til.


DNS fungerer som en telefonbog for internettet.

Når folk skriver domænenavne, som google.com

eller Facebook.com, i adresse-linjen ( *URL’en ) i webbrowsere,

finder DNS den rigtige IP-adresse.



Når først DNS-serveren finder den korrekte IP-adresse,

browsere adressen og bruger den til at sende data til Content Delivery Network (CDN)

edge-servere eller origin-servere.


Når dette er gjort, kan oplysningerne på hjemmesiden tilgås af brugeren.


DNS-serveren starter processen ved at finde den tilsvarende IP-adresse til et websteds *URL


*URL = (Uniform ressource locator).




Hvordan virker DNS?

I en sædvanlig DNS-forespørgsel (query) skal den URL

som brugeren har indtastet, gå gennem 4 servere,

for at IP-adressen kan angives.


De 4 servere arbejder med hinanden.

For at få den korrekte IP-adresse til klienten.


  |----------------------------------------------------------|


De inkluderer:


1. DNS Recursor:

DNS Recursor’en... Som også referes som en DNS resolver.

Der modtager forespørgslen ( query ), fra The DNS Client.


Derefter kommunikerer den med andre DNS servere.

For at finde den rigtige IP-adresse.


Efter at Resolver'en har hentet anmodningen fra klienten.

Fungerer resolver'en som en klient selv.


Mens den gør dette, laver den forespørgsler, ( querries )

der sendes til de 3 andre DNS servere:


  • Root Name-servere.

  • Top-Level Domain - ( TLD ).

  • Authoritative Name-servere.

  |----------------------------------------------------------|


2. Root Name-servere:

Root Name-servere... Er udpeget ( designated ),

til internettets ( DNS Root Zone ).


Dens opgave er at besvare forespørgsler.

Sendt til den, omkring ( records ) registreringer i ( Root zone ).


Den besvarer anmodninger ved at sende en liste tilbage.

Over de ( Authoritative name-servere ), der følger med den korrekte TLD.


  |----------------------------------------------------------|


3. TLD Name-servere:

TLD-name-servere... beholder IP-adressen på domænet.

På ( second-level domain ) indeholdt ( Contained ) i TLD-navnet.


Det frigiver derefter webstedets IP-adresse,

og sender forespørgslen ( query ) til ( domain’s name-servere )


  |----------------------------------------------------------|


4. Authoritative nameservers:

Authoritative nameservers... Er det, der giver dig det rigtige svar.

På din DNS-forespørgsel ( query ).


Der er 2 typer Authoritative nameservers:

  • Master server / Primary name-server.
    • Beholder de originale kopier af zone records.

  • Slave server / Secondary name-server.
    • Er en nøjagtig kopi af masterserveren-






Ekstra - DNS forklaring:



Authoritative DNS Servers vs. Recursive DNS Servers:

- Hvad er forskellen? -

Authoritative nameservers opbevarer oplysninger om DNS records.

En Recursive server fungerer som en ”middle-man.”

Placeret mellem den autoritative server, og slutbrugeren. ”End user”


For at nå name-serveren, skal den Recursive server "gentage" ( recurse ) gennem ”DNS tree”

For at få adgang til domænets registreringer. ( the domain’s records. )


  |----------------------------------------------------------|


Authoritative DNS Server


For at bruge telefonbogs-analogien...

Man skal tænke på IP-adressen, som telefon-nummeret og personens navn som websitets URL.

Autoritative DNS-servere har en kopi af "telefonbogen"


Der forbinder disse IP-adresser med deres tilsvarende domænenavne. ”corresponding” domain names.

De giver svar på de forespørgsler, ( querries ) der sendes af recursive DNS name-servere.

Og giver oplysninger om, hvor man kan finde specifikke websites.


De tildelte svar har IP-adresserne på de domains,

Der er involveret i forespørgslen. ( Query )



Authoritative DNS-servere er ansvarlige for specifikke regioner.

Såsom et land, en organisation eller et lokalt område.


Uanset hvilken region der er dækket.

udfører en autoritativ DNS server, 2 vigtige opgaver.



1st:
Beholder serveren lister over domænenavne ”domain names”

Og de IP-adresser, der følger med dem.


2nd:
Derefter svarer serveren på anmodninger.

Fra den rekursive DNS-server

Vedrørende den IP-adresse,

der svarer til et domænenavn. ”domain name”


Når den rekursive DNS-server får svaret,

sender den disse oplysninger tilbage,

til den computer, der anmodede om det.


Computeren bruger så de her infomrationer.

til at forbinde forbindelse til IP-adressen,

og brugeren får lov til at se hjemmesiden.


  |----------------------------------------------------------|


Recursive DNS Server


Efter en bruger har indtastet en URL i deres webbrowser,

Gives den her URL til den rekursive DNS-server.


Den rekursive DNS-server undersøger derefter sin cache hukommelse ”cache-memory”

For at se, om IP-adressen for URL'en allerede er gemt.( stored / opbevaret )


Hvis IP-adresseoplysningerne ( IP address information ) allerede eksisterer,

vil den rekursive DNS-server sende IP-adressen til browseren.


Brugeren kan derefter se den hjemmeside, som de har indtastet URL'en til.



På den anden side, hvis den recursive DNS-server.

.Ikke finder IP-adressen, når den søger i sin hukommelse, ”memory”

Vil den fortsætte gennem processen med at hente IP-adressen til brugeren.


Den recursive DNS-servers næste trin.

Er at gemme ( oplagre / store ) IP-adressen i et bestemt tidsrum.


Denne tids-periode er defineret af den person.

Der ejer domain'et ved hjælp af en indstilling,

Der kaldes time to live (TTL).





Browser - DNS Caching:


Ens Operativsystem (OS), der anvendes på ens enhed,

( stored ) gemmer DNS resource records ved brug af caching.


Caching forhindrer redundans, ( udnødvendigheder )

når nogen forsøger at gå til et websted.


Dette reducerer til gengæld den tid, det tager at komme til websitet.


Hvis den enhed, man anvender, ”for nylig,”

gik til den side, den forsøger at få adgang til,

kan IP-adressen leveres af cachen.


På den her måde...

Kan website anmodningen fuldføres, uden at involvere DNS-serveren.


DNS-cachen hjælper derved med at ”streamline”

DNS lookup process'en - som ellers ville være nødvendig

For at link et domain-name til en IP-adresse.

Dette gør processen med at komme til websitet meget hurtigere.





OS - DNS Caching


Operativ-systemerne på mange forskellige enheder.

Er i stand til at vedligeholde en lokal kopi af DNS lookups.


Dette gør det muligt for operativsystemet hurtigt.

få de oplysninger, det skal bruge for at løse.

”Resolve” URL'en til den korrekte IP-adresse.





Hvad er typerne af DNS Queries?


Under ”DNS lookup process...”

Udføres 3 forskellige slags forespørgsler. ( Queries )


Forespørgslerne ( The querries ) kombineres

For at optimere opløsningen ( resolution )

Af DNS. hvilket sparer tid.


  • Recursive query

  • Iterative query

  • Non-recursive query

What is QoS in Networking?



Hvad er IPv4?


IPv4 er en IP-version, der er meget brugt til at identificere enheder,

på et netværk ved hjælp af et addressing-system.


IPv4 var den første version af IP.

Der blev implementeret til produktion i ARPANET i 1983.


IPv4 bruger et 32-bit ”address scheme”

Til at gemme 2^32 adresser, hvilket er mere end 4 milliarder address.


Det anses som den primære internet-protocol

og holder 94% af internet-trafikken.


  |----------------------------------------------------------|


Følgende er funktionerne i IPv4:

  • Forbindelses-fri protokol.

  • Tillad oprettelse af et simpelt virtuelt kommunikation-layer over diversified enheder.

  • Kræver mindre hukommelse, og lettere at huske addresses.

  • Allerede supported protokol, af millioner af enheder.

  • Tilbyder video-biblioteker og konferencer.


Hvad er IPv6?


IPv6 er den seneste version af internet-protokollen...

Denne nye IP-address version er ved at blive ”deployed.”

For at opfylde behovet, for flere internet address.


Det havde til formål at løse.

”Resolve” problemer, der er forbundet med IPv4.


Med 128-bit ”address space”

Tillader det 340 undebillion unikke "address space".


IPv6 kaldes også IPng - (Internet Protocol next generation).



Internet Engineer Taskforce startede det i begyndelsen af 1994.

Designet og udviklingen af denne suite kaldes nu IPv6.


  |----------------------------------------------------------|


Følgende er funktionerne i IPv6:

  • Hierarkisk addressing og routing infrastruktur

  • *Stateful og *Stateless konfiguration

  • Support til service-kvalitet (QoS)

  • En ideel protokol til interaktion med "neighbouring node interaction"
    • Internet Control Message Protocol - for IPv6 - (ICMPv6)


*QoS = Quality of service (QoS)

Er brugen af mekanismer eller teknologier, der virker på et netværk
for at kontrollere trafik og sikre ydeevnen af kritiske applikationer
med begrænset netværkskapacitet.

Det gør det muligt for organisationer at justere deres overordnede netværks-trafik
Ved at prioritere specifikke højt-ydende applikationer.

  |----------------------------------------------------------|


*Stateful

Stateful applikationer og processer er dog dem,
der kan vendes tilbage til igen og igen - Såsom: netbank eller e-mail.

De udføres i sammenhæng med tidligere transaktioner.
Og den aktuelle transaktion kan blive påvirket af.
Hvad der skete under tidligere transaktioner.

Af disse grunde bruger statelige apps de samme servere.
Hver gang de behandler en anmodning - Fra en bruger.

  |----------------------------------------------------------|


*Stateless

En stateless proces eller ansøgning kan forstås isoleret.
Der er ingen lagret viden om eller reference til tidligere transaktioner.
Hver transaktion udføres som fra bunden for første gang.

Stateless applikationer leverer én tjeneste eller funktion.
Og bruger Content Delivery Network - (CDN), web eller print-servere.
Til at behandle disse kortsigtede anmodninger.


Hvad er forskellen imellem IPv4 & IPv6


IPv4 og IPv6 er begge IP-adresser, der er binære tal.

IPv4 er et 32-bit binært tal, og IPv6 er en 128-bit binært tal addresses

IPv4-adresser er adskilt af punktum, ( . ) mens IPv6-adresser er adskilt af kolon. ( : )


Begge IP-adresser bruges til at identificere maskiner, der er tilsluttet et netværk.

I princippet ligner de næsten hinanden, men de er forskellige i, hvordan de fungerer.


IPv4 og IPv6 kan ikke kommunikere med andre.

Men kan eksistere sammen på det samme netværk.

Det er kendt som Dual Stack.


IPv4 er den fjerde version af Internet Protocol (IP).

Mens IPv6 er den seneste version af Internet Protocol.

Derfor er IPv6 mere avanceret, sikker og hurtigere - Sammenlignet med IPv4.






Minor overview:

  • IPv4 er en 32-bit IP-adresse - Hvorimod IPv6 er en 128-bit IP-adresse.

  • IPv4 er en numerisk addressing method - Hvorimod IPv6 er en alfa-numerisk addressing method.

  • IPv4 binære bit er adskilt af en prik(.) - Hvorimod IPv6 binære bit er adskilt af et kolon(:).

  • IPv4 tilbyder 12 ”*Header fields” - Hvorimod IPv6 tilbyder 8 ”*Header fields.”

  • IPv4 support broadcast - Hvorimod IPv6 ikke supporter broadcast.

  • IPv4 har ”*Checksum fields” - Mens IPv6 ikke har ”*Checksum fields.”

  • Når man sammenligner - IPv4 og IPv6.

    • IPv4 - Support - VLSM (Variable Length Subnet Mask - Diagram).

    • IPv6 - Support ikke - VLSM.


  • ARP ( Address Resolution Protocol ) - NDP ( Neighbour Discovery Protocol )

    • IPv4 bruger ARP - Til at kortlægge ”map” til MAC-adresse.

    • IPv6 bruger NDP - Til at kortlægge ”map” til MAC-adresse.


*Header fields. = Overskrifts-felter.


*Checksum fields. = Kontrolsum-felter.

Windows-operativsystemet giver sin bruger et kraftfuldt værktøj - Dvs. kommandoprompt. ( CMD )

Som giver os mulighed for at få adgang til og konfigurere system-indstillinger og data.


De her netværks-kommandoer kan være nyttige...

Når der er behov for at konfigurere…  Eller fejlfinde netværks-indstillingerne i vores system.



Hvad er Network Commands:


Hvert system er forbundet til adskillige forskellige netværk,

og systemer gennem interne eller eksterne netværks-forbindelser.

De her netværks-indstillinger løber ofte ind i problemer og påvirker systemets funktioner.


De her netværks-problemer kan løses ved hjælp af ’Network-commands’.


De her kommandoer er specielt designet til at fejlfinde netværks-problemer.


Med en minimal overblik ved hjælp af Windows kommandoprompt værktøj. ( CMD.exe )


Indenfor Network commands – Er der en alt for stor liste af network commands at gennemgå…


Her er et lille udvalg...


Af de mest grundlæggende netværks-kommandoerne:



IPCONFIG:


ipconfig netværks-kommandoen giver et omfattende overblik.

Over almen oplysninger vedrørende IP-adresse konfigurationen.

For den enhed, vi arbejder på i øjeblikket.


ipconfig kommandoen har en vis input variation udover 'ipconfig' kommando.

Der er rettet mod specifikke system-indstillinger eller data.


Viser alle de aktuelle TCP/IP-netværks konfigurationsværdier.

Opdaterer *DHCP ( Dynamic Host Configuration Protocol ).

og *Domain Name System ( DNS ) indstillinger.


Den her kommando bruges hovedsageligt til at se IP-adresserne.

På de computere, der er konfigureret til at hente deres IP-adresse automatisk.



*DHCP = ( Dynamic Host Configuration Protocol )
Server / Funktion - Der deler ud / sender ud IP adresser - Til netværkets enheder / clients.

*DNS = ( Domain Name System )
Server / Funktion - Der tildeler et navn til en IP-adresse.


  |----------------------------------------------------------|



Input kommandoen: – ipconfig

Beksrivelse:  Display IP-adresserne - Subnet Mask - Default Gateway - For alle netværk adapters



Commands: Beskrivelse:

ipconfig/renew [Adapter] Renews - DHCP konfiguration - For alle adapters - ( Hvis en adapter er inkluderet )
ipconfig/release [Adapter] Sender en DHCP(RELEASE) besked til ens DHCP server.
For at 'Release' Frigøre den nuværende DHCP konfiguration.
Og discard ( Fjerne ) IP adresse konfiguration. - ( Hvis en adapter er inkluderet )
ipconfig/flushdns Flushes ( Renser ) - Og resets ( Omstarter ) indholdet af ens DNS client resolver cache.


  |----------------------------------------------------------|



ipconfig - promt - eksempler:

PING


Ping-kommandoen er en af de mest udbredte kommando værktøj.

Siden den giver brugeren mulighed - For at kontrollere forbindelsen.

Imellem vores system - Og en anden host / client / enhed.


Den her kommando sender 4 'test packets = *ICMP' til destination hosten.

For at kontrollere, om den modtager dem med en succes rate. ( 100% = 4 recieved ) - ( Hvis ja )...

Kan man kommunikere med destination hosten.


Men i tilfælde af, at pakkerne ( packets ) ikke er blevet modtaget. ( 100% = 4 losts )

Betyder det - At der ikke kan etableres kommunikation med destination hosten.


Ping-kommandoen bruges til at teste forbindelsen mellem to hosts ( enheder )

Den sender ICMP echo-requests messages til destinationen.

Destination hosten svarer med ICMP-reply message.


Hvis ping-kommandoen får et reply fra destination hosten.

Viser den the reply sammen med round-trip times info.


Hvis man angiver hostname som et argument.

Bruger ping-kommandoen den konfigurerede DNS client service.

Til at... Automatisk oversætte hostname til IP-adressen.



  |----------------------------------------------------------|



Input kommandoen: – ping


Ping destination - [ Hosten's IP ] eller [ DNS Navn ]


Input kommandoen: ( DNS ) – ping google.com


Input kommandoen: ( IP ) – ping 8.8.8.8

TRACERT


Kommandoen bruges til at spore ruten under transmissionen af datapakken.

Over til destination hosten - Giver os også "hop"-counts under transmissionen.


Ved at bruge antallet af hop og hop IP-adressen.

Kan vi fejlfinde ( troubleshoot ) netværks-problemer.

Og identificere problemets punkter i ruten - Under overførslen af datapakken.


Den her kommando bruges til at diagnosticere path-relaterede problemer.

På et IP-netværk... Der vil 'Routen' veklse IP-pakker - Imellem kilden ( source ) og destinationen.

De tager IP-pakker - Fra the source host - Og videresender dem i en rækkefølge ( sequence ) - Indtil de når destination hosten.


Rækkefølgen af routere - Imellem kilden ( source ) og destinationen - Er kendt som: ( The Path ).


En 'path' består af alle routere - I en sekvens - Som IP-pakkerne der er sendt.

Fra source host der gennemstrømmer ( traverse ) - For at nå destination host.


Kommandoen printer the path.

Hvis alle routere på the path - Er funktionelle - Printer den her kommando hele the path.

Hvis en router er down på the path - Printer den her kommando the path op til den 'sidste operationelle router.



  |----------------------------------------------------------|



Input kommandoen: – tracert


Tracert destination - [ Destination Navn ] eller [ IP adresse ]


Input kommandoen: ( DNS ) – tracert google.com


Input kommandoen: ( IP ) – tracert 8.8.8.8




Ekstra valgmuligheder af tracert kommandoen:


Commands: Beskrivelse:
tracert -d Man kan ikke 'resolve' ( Ændre ) - På IP-adresserne af 'Intermediate Routere' navne.
tracert -h Angiver det maksimale - Antal hop ( routere )
Der skal søges på 'The Path'
Default: 30 hops.
tracert -w Angiver antallet af tid i mm.
Der venter på en reply message fra routeren.
Hvis den ikke modtages inden for time-out, er display'ed med en stjerne (*) - ( asterisk )
Default time-out er 4000 (4 seconds).

NETSTAT


Kommandoen viser en oversigt - Over alle netværks forbindelserne i enheden.


Tabellen viser detaljer:

Om forbindelses protokollen,

Adressen og netværkets nuværende tilstand.


Den her kommando viser aktive forbindelser, porte, som computeren lytter til.

Ethernet-statistik - IP-routing tabellen - IP-statistik.


Outputtet af den her kommando - Er organiseret i rækker og kolonner.

Hver række repræsenterer - En ny forbindelse eller en indgang i outputtet.

Indeholder 4 kolonner - De her kolonner giver følgende oplysninger om rækken ( Row ).


  • Proto:  Den her kolonne viser navnet på protokollen (+TCP eller +UDP).


  • Local address:  Den her kolonne viser IP-adressen på den lokale computer og det port-nummer, der anvendes.

    • Hvis porten endnu ikke er etableret - Vises port-nummeret som en stjerne (*).


  • Foreign Address:  Den her kolonne viser IP-adressen og port-nummeret på den remote-computer, som porten er tilsluttet


  • State:  Den her kolonne viser status for forbindelsen.




Ekstra valgmuligheder af netstat kommandoen:


Commands: Beskrivelse:
netstat -a Displays alle aktive TCP forbindelser og TCP og UDP portene på computeren der afhører med ( listening. )
netstat -e Displays Ethernet statistik, såsom nummeret af bytes og packets der sendt og modtaget.
netstat -n Displays aktive TCP forbindelser.
Hvorledes, adresserne og port numre er udtrykt numrerisk
Og intet forsøg er skab til at fastlægge navne.
netstat -o Displays aktive TCP forbindelser og inkludere processen ID (PID) for hver forbindelse.
netstat -p Viser forbindelserne for protokollen specifikation af Protokollen.
I det her eksempel, kan protokollen være TCP, UDP, tcpv6, eller udpv6.
netstat -s Displays statistik af protokollen.
Ved default, er statistikken vist for TCP, UDP, ICMP, og IP protokoller.
netstat -r Display'er indholdet af IP routing tabellen.

TCP = Transport Control Protocol

UDP = User Datagram Protocol

ICMP = Internet Control Message Protocol

ARP - ( Address Resolution Protocol )


Kommandoen bruges til at få adgang - Til mapping structure - Af IP-adresser til MAC-adressen.

Det her giver os en bedre forståelse - Af transmissionen af packets i network channel.


For at sende IP-pakker - Skal en computer bruge to adresser.

De her adresser er - MAC-adressen og IP-adressen.

MAC-adresse:  Er NIC'ens fysiske adresse eller hardware-adresse ( Fysisk ID )

IP-adresse:  Er NIC's logiske adresse eller software-adresse. ( Logisk ID )


Hvis en computer kender destination computerens IP-adresse.

Men den ikke kender destination computerens MAC-adresse

Bruger den ARP-protokollen til at kende destination computerens MAC-adresse.



ARP-Protokollen - Broadcasts en givende IP-adresse, over et lokalt network ( LAN ).

Den tilsvarende host - Respons på the broadcast med sin MAC-adresse.

For at undgå gentagelse - Gemmer ( stores ) ARP svaret i en tabel - Kendt som: ARP-table.

ARP vedligeholder en separat ARP-table - For hvert NIC.



Input kommandoen: – arp





  |----------------------------------------------------------|



Som default - Viser kommandoen ARP-protokollen - For det aktive NIC.

Hvis der er installeret flere NIC's på computeren.


Kan man anvende input -a med denne kommando.

Hvis input -a anvendes - Viser ARP-kommandoen - For alle ARP-tabeller.



Input kommandoen: – arp -a



  |----------------------------------------------------------|



For at vise ARP-cache-entry for en specifik IP-adresse

Skal man angive IP-adressen med -N inputtet.


Kan man bruge inputtet -a med denne kommando.

Hvis inputtet -a anvendes - Viser ARP-kommandoen alle ARP-tabeller.


Viser følgende kommando ARP-cache-table for den interface.

F.eks - Der er tildelt med IP-adressen 192.168.42.171.



Input kommandoen: – arp -a -N 192.168.42.171.
